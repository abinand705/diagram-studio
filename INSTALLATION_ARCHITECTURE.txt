SECTION 2: DETAILED INSTALLATION STEPS
================================================================================

STEP 1: PREPARE ENVIRONMENT
==========================

1A. Create Project Directory
----
Windows:
  md C:\Users\YourName\Desktop\smart-diagram
  cd C:\Users\YourName\Desktop\smart-diagram

Mac/Linux:
  mkdir ~/Desktop/smart-diagram
  cd ~/Desktop/smart-diagram

1B. Open in VS Code
----
Option 1 - Via UI:
  - Open VS Code
  - File → Open Folder
  - Select smart-diagram folder

Option 2 - Via Command:
  code .

1C. Set Up Git (Optional but Recommended)
----
  git init
  git config user.name "Your Name"
  git config user.email "your@email.com"

STEP 2: BACKEND SETUP
====================

2A. Create Backend Directory
----
In VS Code Terminal:
  mkdir backend
  cd backend

2B. Initialize NPM Project
----
  npm init -y

Output:
  This will create package.json with defaults

2C. Install Dependencies
----
  npm install express cors dotenv

Output:
  - Creates node_modules/ folder
  - Updates package.json

2D. Create Environment File
----
Method 1 - Via Terminal:
  echo OPENAI_API_KEY=sk-your-key > .env

Method 2 - Via VS Code:
  - Right-click on backend folder
  - New File
  - Name: .env
  - Add: OPENAI_API_KEY=sk-your-key

2E. Create Server File
----
  - Right-click on backend folder
  - New File
  - Name: server.js
  - Copy entire server.js code from outputs folder
  - Paste into the file
  - Ctrl + S to save

2F. Install Production Dependencies (Optional)
----
For deployment, also install:
  npm install dotenv

2G. Test Backend
----
In Terminal:
  node server.js

Expected Output:
  🚀 Smart Diagram Generator running on port 5000
  📍 API: http://localhost:5000

Verification:
  - Open browser: http://localhost:5000
  - Should see: {"message":"Smart Diagram Generator API is running 🚀"}

STEP 3: FRONTEND SETUP
=====================

3A. Return to Project Root
----
In Terminal (Ctrl + C to stop server first):
  cd ..

3B. Create React Application
----
  npx create-react-app frontend

⏳ WAIT: This takes 2-3 minutes...

Expected Output:
  Happy hacking!

3C. Navigate to Frontend
----
  cd frontend

3D. Install React Flow Library
----
  npm install @xyflow/react

Output:
  Added 20 packages

3E. Replace App.js
----
  - Open frontend/src/App.js
  - Delete all existing code
  - Copy entire App.js from outputs folder
  - Paste into the file
  - Ctrl + S to save

3F. Test Frontend
----
  npm start

Expected Output:
  Compiled successfully!
  You can now view frontend in the browser.

Browser should automatically open:
  http://localhost:3000

STEP 4: VERIFY BOTH RUNNING
===========================

Terminal Setup:
  Terminal 1: Backend (node server.js running)
  Terminal 2: Frontend (npm start running)

Or open separate Terminal windows:
  - Window 1: cd backend && node server.js
  - Window 2: cd frontend && npm start

Browser:
  - Open http://localhost:3000
  - Should see diagram generator interface

SECTION 7: COMMON ISSUES & SOLUTIONS
================================================================================

ISSUE 1: Port Already in Use
Error: "Error: listen EADDRINUSE: address already in use :::5000"

Solution A - Kill Process:
  Windows:
    netstat -ano | findstr :5000
    taskkill /PID [PID] /F
  
  Mac/Linux:
    lsof -i :5000
    kill -9 [PID]

Solution B - Use Different Port:
  In server.js change:
    app.listen(5000) → app.listen(5001)
  
  In App.js fetch calls change:
    localhost:5000 → localhost:5001

ISSUE 2: Module Not Found
Error: "Cannot find module 'express'"

Solution:
  cd backend
  npm install express cors dotenv

ISSUE 3: CORS Error
Error: "Access to XMLHttpRequest blocked by CORS policy"

Solution:
  Ensure server.js includes:
    const cors = require("cors");
    app.use(cors());

ISSUE 4: React App Won't Start
Error: "React script not found"

Solution:
  cd frontend
  npm install
  npm start

ISSUE 5: API Returns 404
Error: "Cannot POST /generate-er"

Solution:
  - Verify backend is running (port 5000)
  - Check endpoint spelling in frontend
  - Ensure server.js has the route defined

ISSUE 6: Drag-and-Drop Not Working
Error: No error, but components won't drag

Solution:
  - Make sure backend is running
  - Try refreshing browser
  - Open DevTools (F12) for errors
  - Check that React Flow nodes are properly configured

ISSUE 7: Export Button Not Working
Error: Download fails silently

Solution:
  - Ensure you have nodes on canvas
  - Check browser console (F12)
  - Verify ReactFlow ref is properly set
  - Try different browser

ISSUE 8: Template Won't Load
Error: "Failed to load template"

Solution:
  - Backend must be running
  - Test: curl http://localhost:5000/templates
  - Check network tab (F12) for errors
  - Verify template ID is correct

SECTION 8: PERFORMANCE OPTIMIZATION TIPS
================================================================================

FRONTEND OPTIMIZATION:

1. Lazy Load Components:
   - Load templates only when needed
   - Split large components

2. Memoization:
   - Wrap components with React.memo()
   - Prevents unnecessary re-renders

3. Event Debouncing:
   - Debounce drag events
   - Throttle wheel zoom

4. Code Splitting:
   - Dynamic imports for large libraries
   - Separate route chunks

BACKEND OPTIMIZATION:

1. Caching:
   - Cache template data
   - Cache frequently used endpoints

2. Database Indexing:
   - Index diagram ID
   - Index user queries

3. Compression:
   - Gzip responses
   - Minify JSON

4. Connection Pooling:
   - Database connection reuse
   - Reduce overhead
