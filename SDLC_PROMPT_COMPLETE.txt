PHASE 1: PLANNING & REQUIREMENTS
================================================================================

1.1 PROJECT OBJECTIVE
---------------------
Build a professional web-based diagram generator that allows users to create
Entity-Relationship (ER) and Data Flow Diagrams (DFD) without login/registration.

Key Features:
- Drag-and-drop component system similar to draw.io
- Support for ER and DFD diagrams
- AI-powered generation from natural language
- Export functionality (PNG, JSON)
- Template system for common scenarios
- Validation and suggestions engine

2. Junior Developers

3.2 DATA MODELS
---------------

DIAGRAM STRUCTURE:
{
  id: string,
  name: string,
  type: "ER" | "DFD",
  nodes: [
    {
      id: string,
      type: "entity" | "relationship" | "process" | "datastore" | "externalentity",
      position: { x: number, y: number },
      data: {
        label: string,
        attributes?: string[],
        properties?: object
      }
    }
  ],
  edges: [
    {
      id: string,
      source: string,
      target: string,
      label?: string,
      type?: "1:1" | "1:N" | "M:N"
    }
  ],
  createdAt: timestamp,
  updatedAt: timestamp
}

NODE TYPES:
1. Entity (ER)
   - label: string
   - attributes: string[]
   - primaryKey: string
   - color: string

2. Relationship (ER)
   - label: string
   - type: "1:1" | "1:N" | "M:N"

3. Process (DFD)
   - label: string
   - number: string (e.g., "2.1")

4. DataStore (DFD)
   - label: string
   - type: "Database" | "File" | "Cache"

5. ExternalEntity (DFD)
   - label: string
   - role: string

3.3 API ENDPOINTS
-----------------

CORE ENDPOINTS:

Diagram Management:
  POST   /generate-er          Generate ER diagram
  POST   /generate-dfd         Generate DFD diagram
  POST   /save-diagram         Save diagram
  GET    /diagrams             Get all diagrams
  GET    /diagrams/:id         Get specific diagram
  DELETE /diagrams/:id         Delete diagram

Templates:
  GET    /templates            Get all templates
  GET    /templates/:id        Get specific template

Validation:
  POST   /validate             Validate diagram design

Export:
  POST   /export/json          Export as JSON
  POST   /export/png           Export as PNG (future)
  POST   /export/pdf           Export as PDF (future)

3.4 DATA FLOW
-------------

User Input Flow:
1. User types description or drags components
2. Frontend collects input (text/components)
3. Sends to backend via POST
4. Backend processes (AI or mock)
5. Returns structured data (JSON)
6. Frontend renders diagram
7. User can export or save

Drag-and-Drop Flow:
1. User drags component from sidebar
2. onDragStart captures component type
3. onDragOver allows drop zone
4. onDrop creates new node at position
5. React Flow re-renders
6. Node appears on canvas
7. User can connect to other nodes

Export Flow:
1. User clicks export button
2. Frontend captures canvas state
3. Converts to required format
4. Creates download blob
5. Triggers browser download
6. File saved to user's device
